2026-04-17: observability를 2차 개편했다. 사용자는 CPU 상세 표보다 실제 운영 증거를 더 원했고, pod 레벨까지만 보면 충분하다고 명시했다. Backend는 logs/targets, logs/stream API를 추가했다. logs/targets는 app.kubernetes.io/name 기준 pod 목록과 각 container의 ready/restart/default 정보, metrics.k8s.io 기반 현재 cpu/memory usage, pod spec resources.requests/limits를 결합해 resourceStatus(cpuUsageCores, cpuRequestCores, cpuLimitCores, memoryUsageMiB, memoryRequestMiB, memoryLimitMiB 및 utilization)을 돌려준다. logs/stream은 podName/containerName 지정형 SSE endpoint이며 viewer도 조회 가능하다. Frontend App.tsx 관측 탭은 metric summary 카드만 남기고 MetricDataTable을 제거했다. 대신 pod Select + container Select + live status badge(연결 중/실시간 수신 중/스트림 종료/오류) + 실시간 로그 pre + 선택 container의 CPU/메모리 사용량 대비 request/limit 카드로 바뀌었다. 사용자 선호: observability는 설명용 chart보다 실제 pod/container 기준 로그와 할당 대비 사용량이 먼저 보이는 구조여야 한다. 관련 문서: docs/user-feedback-log.md FB-039, docs/internal-platform/openapi.yaml logs/targets/logs/stream 스키마.
