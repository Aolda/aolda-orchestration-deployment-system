2026-04-17: 사용자 신뢰 강화를 위해 애플리케이션 운영 센터의 관측 탭에 실제 pod/container 로그를 노출하는 기능을 추가했다. Backend는 GET /api/v1/applications/{applicationId}/logs API로 최근 pod의 primary container 로그를 읽고, viewer 권한도 조회 가능하다. Kubernetes 구현은 labelSelector app.kubernetes.io/name={appName} 기반 pod 목록 -> running 우선/최신순 정렬 -> sidecar(istio-proxy/linkerd-proxy/vault-agent) 제외 primary container 선택 -> tailLines/timestamps 옵션으로 최근 로그 수집 구조다. Frontend는 App.tsx 관측 탭에 최근 컨테이너 로그 카드로 pod명, 컨테이너명, phase, ready, restartCount, 로그 본문을 보여준다. 피드백 기록은 docs/user-feedback-log.md FB-035다. 사용자 선호: 메트릭/Sync만으로는 신뢰하지 않고 실제 런타임 증거(로그/이벤트)가 보여야 한다.
