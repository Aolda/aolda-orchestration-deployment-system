2026-04-18: Added repository polling visibility for repo-backed applications. Backend now has AODS_REPOSITORY_POLL_INTERVAL (default 5m) in core.Config and logs it at startup. application.Service carries an in-memory RepositoryPollTracker updated by AutoUpdatePoller on attempt/success/failure. GET /api/v1/applications/{applicationId}/sync-status now returns repositoryPoll with enabled, intervalSeconds, nextScheduledAt, lastCheckedAt, lastSucceededAt, lastResult, lastError, and source. Frontend App status drawer shows 최근 저장소 확인 / 다음 저장소 확인 cards plus a polling summary alert derived from syncStatus.repositoryPoll. Tests added in backend/internal/application/poll_tracker_test.go and backend/internal/application/service_test.go.
