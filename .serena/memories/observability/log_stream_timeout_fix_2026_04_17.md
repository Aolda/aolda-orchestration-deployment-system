2026-04-17: Kubernetes pod log SSE stream에서 `read kubernetes api stream: context deadline exceeded (Client.Timeout or context cancellation while reading body)`가 발생한 원인은 backend/internal/kubernetes/real.go 의 apiClient.HTTPClient timeout이 일반 K8s API 조회용 timeout(예: 5s)을 로그 stream에도 그대로 적용했기 때문이다. fix: StreamText는 c.HTTPClient를 그대로 쓰지 않고 Timeout=0 으로 복제한 streamingHTTPClient()를 사용한다. 일반 GetJSON/GetText timeout은 유지한다. 회귀 테스트: TestPodLogReaderStreamIgnoresHTTPClientTimeoutForLongLivedLogs. 사용자 관측: 프론트에서 stale pod/container 재선택 로직도 별도로 넣어 두었으므로, 앞으로 로그 실패 시 먼저 timeout vs stale target을 구분해서 볼 것.
