White-screen bug in frontend/src/App.tsx network exposure section was caused by reading event.currentTarget.checked inside the functional state updater for Mantine Switch onChange handlers. After React processed the event, currentTarget became null and toggling LoadBalancer (and potentially mesh) crashed the whole App with `Cannot read properties of null (reading 'checked')`. Fix: capture `const checked = event.currentTarget.checked` before calling `setApplicationNetworkDraft`, then use the captured primitive inside the updater for both `meshEnabled` and `loadBalancerEnabled`. Reproduced in browser via Playwright by toggling the LoadBalancer switch in app drawer > 운영 규칙; after fix, switch toggles and page stays rendered.
