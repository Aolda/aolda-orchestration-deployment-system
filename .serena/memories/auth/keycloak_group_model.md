2026-04-16 업데이트: AODS는 Keycloak group hierarchy 재사용이 기본이지만, OIDC parser가 이제 resource_access.*.roles에 대해 bare role과 client-qualified role을 둘 다 수집한다. 예: resource_access.account.roles=[manage-account] 이면 user.Groups에 manage-account 와 account:manage-account 가 함께 들어온다. 전역 admin을 role 기반으로 줄 때는 bare manage-account 보다 account:manage-account 를 AODS_PLATFORM_ADMIN_AUTHORITIES / VITE_AODS_PLATFORM_ADMIN_AUTHORITIES 에 넣는 편이 안전하다. 단 screenshot처럼 manage-account 가 default roles에 포함돼 있으면 일반 사용자도 admin이 될 수 있으므로 운영 기본값으로 켜지 말 것. 자세한 내용은 docs/keycloak-group-auth-model.md 참고.
