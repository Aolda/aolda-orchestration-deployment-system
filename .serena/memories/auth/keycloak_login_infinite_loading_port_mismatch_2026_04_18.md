Root cause for post-login infinite loading in local dev was frontend origin drift, not bootstrap API failure. Vite had silently moved from localhost:5173 to 5174, while OIDC redirect URI in frontend/.env.local and backend allowed origin remained pinned to http://localhost:5173. Fix applied: frontend/vite.config.ts now uses host=localhost, port=5173, strictPort=true so dev server fails fast instead of auto-shifting ports; frontend/src/auth/oidc.ts now validates loopback redirect origin against window.location.origin and throws a clear error when VITE_AODS_OIDC_REDIRECT_URI does not match the current frontend origin. Verified after restart: emergency login renders app shell successfully on http://localhost:5173, and Keycloak button redirects to https://sso.aoldacloud.com/realms/aolda/... with redirect_uri=http://localhost:5173. Also stopped stale old dev server on 5174 from the same repo to prevent confusion.
