2026-04-18: Investigated user report of infinite loading after login. Local reproduction with emergency login admin/qwe1356@ succeeded, so issue was treated as intermittent bootstrap hang risk rather than deterministic credential failure. Implemented frontend hardening in frontend/src/api/client.ts by wrapping regular API requests with a 15s timeout and converting timeout/network failures into ApiError codes REQUEST_TIMEOUT and NETWORK_ERROR. This prevents full-screen bootstrap loader from hanging forever when /me, /projects, or /clusters stall. Also updated frontend/src/auth/oidc.ts so cached OIDC discovery promise resets on failure; otherwise one failed discovery request poisoned later retries until reload. Added regression tests in frontend/src/api/client.test.ts for timeout and network failure behavior. frontend/.env.local was aligned to admin/qwe1356@ for consistency, though source code already enforces that emergency credential pair.
